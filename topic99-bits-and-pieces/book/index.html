Using Trace to Debug Programs

Importing library and using trace in programs

Objectives

On completion of this lab you should:

  • be able to use trace, traceShow etc. to debug programs

  • be better able to understand the running of programs using debug

Using Trace

  • We look at one function, multby2 which takes a list and multiplies all elements in the list by 2. 

import Debug.Trace

multby2 :: Num a=> [a] -> [a]
multby2 [] = []
multby2 (x:xs) = x*2 : multby2 xs

multby2' :: Num a=> [a] -> [a]
multby2' [] = trace "running on empty" []
multby2' (x:xs) = trace "recursing" x*2 : multby2 xs

--Note use of (Num a, Show a) here - we need to use show. 
multby2'' :: (Num a, Show a)=> [a] -> [a]
multby2'' [] = trace "running on empty" []
multby2'' (x:xs) = trace ("value of x = "  ++ show x)  x*2 : multby2 xs

Exercise 2

The following sort is bugged. Using trace to help you, fix it. 


sort :: Ord a => [a] -> [a]
sort ( x : xs ) = insert x ( sort xs )
insert :: Ord a => a -> [a] -> [a]
insert x [ ] = [ x ]
insert x ( y : ys ) = if  x <= y   then (x : ys)   else y : insert x ( y : ys )
